Welcome!

The best way to use Table Capture is to:

  1. Highlight any text within the table you're interested in and control clickright-click. This tells the extension which table you care about.
    Note: Sometimes highlighting text is disabled. Usually the extension can still figure out which page element you're interested in from where you click.
  2. Select: Table Capture - Launch workshop for selection from the context menu.
    Note: If you can't highlight text, it's okay. A control clickright-click will give you these options. Use the Workshop! Click: Table Capture - Launch workshop.
  3. The Workshop will open and highlight your table.

Sometimes there are lots of <table> elements on a page. Also, sometimes tabular data is displayed in HTML that isn't structured as a <table>. The Table Capture Workshop helps you extract tabular data however it's rendered and even allows capturing paged tables and dynamic tables.

For tables that aren't created with <table> elements:

You may select some text within a table and find that the Table Capture Workshop is only highlighting a cell or a row of the table and it presenting you with these buttons:

You'll need to use the Select parent element and Select child element buttons to find the page element that wraps all the repeating elements of the table or list.

You can see how this is done with this tutorial video: https://www.youtube.com/watch?v=_ZQsXgGfOTM

Excel & CSV

You can customize how your Excel & CSV files are named using a template above. Variables available: $NAME, $DATE, $DOMAIN

For a table on georgemike.com ($DOMAIN) listed as "(4 x 5) name-table" ($NAME): $NAME.$DATE-heart-$DOMAIN → 4_x_5_name-table.-heart-georgemike.csv

The default behavior is to put each table from a batch in its own sheet. With the above enabled, each table will be placed in a single sheet with a blank row separating each table.

Numeric parsing

If selected, when your cell values contain numeric values, the extension will extract those values as plain numbers.

If selected, when your cell values contain money values, the extension will extract those values as plain numbers. For example: $1,249.99 → 1249.99

Copy to Clipboard

If selected, when Table Capture encounters a link, it will extract the text along with the URL in parentheses. For example: Link test (http://www.the-link-referred-to.com)

If selected, when Table Capture encounters an image, nothing will be extracted. If a table cell contains only an image, the cell will be empty when the table is captured.

If selected, when Table Capture encounters an image, it will extract the alt text along with the src in parentheses. For example: Logo (https://www.georgemike.com/images/port.tablecapture.icon.png)

Additionally, if selected, Table Capture will extract icon attributes to help discern between icon-based cell values.

We'll insert a pipe character ( | ) after text in paragraph <p> tags and for <br> tags.

For <pre> and <code> tags, we'll add this little umbrella character ( ☂ ) for line breaks, which will allow you to do a find/replace and tweak accordingly. It's a little silly, but it works...

If selected, empty table rows will be excluded from the captured output.

When selected, only visible page elements will be extracted.

Experimental Features

When enabled, Table Capture will request permission to write directly to Google Drive. This means you won't have to manually paste in data from the clipboard. This also means, you can choose to overwrite an existing sheet.

You can manage your sheets here.

When enabled, Table Capture can search web requests for data you're interested in and turn JSON responses into tabular data.

Note: This is super experimental. To try request searching, once enabled - click the Table Capture logo from the Workshop. Also, George does not recommend leaving this feature enabled - it might interfere with certain websites.

When enabled, functionality that developers can make use of to get more from Table Capture will be available from within the extension.

When enabled, there will no longer be two toggles for paging and dynamic tables.
